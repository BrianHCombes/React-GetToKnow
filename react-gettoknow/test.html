About LR
LR is a supplemental tool for designing the responsive behavior of web pages across all screen sizes.
  • LR is not a framework and won't interfere with your responsive design approach. However, should traditional approaches falter, LR can be called upon.
  • LR works by doing proportional lineation between two defined points. This provides pixel precision sizing on any given screen allowing for optimum screen usage.
  • LR can be used exclusively for all responsive behavior.
  • LR is arguably easier to use than traditional methods with its relatively small API.
LR is a powerful tool to complement your responsive design toolbox. It uses plain old Javascript and works with all CSS and frontend frameworks. In fact, LR can even resolve the limitations you might encounter with traditional responsive tools. LR works by calculating a proportional lineation between two pre-defined points for precise element sizing. Capable of serving as your only tool for all your responsive needs and, in many cases, uses significantly less code and brings more power in a single stroke.


Most notably, LR manages absolute sizing like 'px' as readily as relative sizing like '%','em','vw', and all other available units.

LR allows percentages to be changed.

However, LR can easily be used exclusively to design all responsive aspects of a web page. Under this consideration, LR is structured around a 'response field' approach rather than a mobile first approach. A response field encompasses all screen sizes from large desktop screens to mobile screens simultaneously. So starting with a desktop first approach is as viable as a mobile first approach since you design for both at the same time.


A reasonable element sizing gradation using @media from a desktop screen to a mobile screen would require about 6 @media code blocks. LR does this same gradation but with a smooth lineation from a desktop screen to a mobile screen with a third of the code.

LR is completely untethered to CSS frameworks yet will work well with them. Thus you can use plain old CSS and design your own custom look from complete scratch.

LR dynamically changes CSS values that might otherwise be static. For example: Consider CSS units '%' and 'em'. They are static because they are the same across all screen sizes. These and all other CSS units are dynamically changeable using LR, including 'px'.

Currently, CSS media queries are used to define demarcation points between element sizes to address the static-ness of '%' and 'em' and others. Rather, LR manages this through lineation and offers superior response resolution. For example: if you had a 1500px width difference between a desktop screen and mobile screen, you'd need 1500 media query statements to realize a pixel by pixel resolution. LR accomplishes this with less code than a single media query line.

In fact, speaking of 'px', current response wisdom shuns use of 'px' as a CSS unit due to it's 'non responsive, fixed value'. But LR works with 'px' as readily as with all the other CSS units. In fact LR encourages the use of 'px' as needed. For example: regarding 'px', elements sometimes have no inherited or defined size and thus won't respond to CSS units like "%". However, using pixels (px) requires no inheritance and LR serves up pixel based sizing with precision.

And finally, you can disregard the mobile first mantra, your response and presentation development work can start at the desktop (big screen) until you have the look you like on said big screen. Then simply switch to a mobile size screen via your favorite developer tool and tailor your LR expressions to fit the mobile screen. Once complete, your presentation will look excellent from the desktop to mobile and all possible screen sizes above, below, and in between. You have thus designed to fit the 'response field'.



LR methods returning CSS values
lr.w("args")* Uses the reported viewport** width to establish the proper CSS return.
lr.ww(args) Uses the reported window** width to establish the proper CSS return.
lr.h(args) Uses the reported viewport** height to establish the proper CSS return.
lr.wh(args) Uses the reported window** height to establish the proper CSS return.
lr.r(args) Uses the reported viewport** height/width ratio to establish the proper CSS return.
lr.wr(args) Uses the reported window** height/width ratio to establish the proper CSS return.
* lr.w(args) will likely be your most used LR method.
